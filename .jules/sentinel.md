## 2026-03-01 - [High] Secure Handling of Passwords in Jetpack Compose UI
**Vulnerability:** Sensitive tokens and passwords in Compose UI state (e.g., API keys, Wi-Fi passwords) were using Compose's `PasswordVisualTransformation` but failed to set the underlying keyboard type to Password and disable OS-level auto-correction.
**Learning:** If a text field uses visual transformation for password masking but retains the default keyboard behavior, the device's OS-level dictionary and keyboard autocomplete engine can silently cache the entered secret, exposing it as an autocomplete suggestion later.
**Prevention:** When building text fields for secrets, always set `keyboardOptions = KeyboardOptions(keyboardType = KeyboardType.Password, autoCorrectEnabled = false)` alongside `PasswordVisualTransformation()` to properly disable OS-level predictive text and dictionary caching.
