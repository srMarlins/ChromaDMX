## 2025-02-17 - Added semantics Role to Modifer.clickable where appropriate
**Learning:** Found an accessibility anti-pattern in Jetpack Compose code. `Modifier.clickable` without a `role` does not communicate enough context to accessibility services (e.g. TalkBack). Providing a semantic role (e.g. Role.Button, Role.Checkbox, Role.Tab) helps screen reader users understand what element they are interacting with. Also replacing bare `Modifier.clickable { ... }` with `Modifier.clickable(onClick = { ... }, role = ...)` was needed.
**Action:** When creating custom clickable UI components in Compose (like buttons, dropdown items, switches, list items), make sure to pass the appropriate `role` to `Modifier.clickable`. For example, `Modifier.clickable(onClick = { ... }, role = Role.Button)` for generic buttons, `Role.Checkbox` or `Role.Switch` for toggles, etc. This is essential for a screen reader to properly announce the component's interactive type.
